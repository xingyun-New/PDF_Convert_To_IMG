# PDFtoJPG

Windows 下**免安装**的 PDF 批量转图片工具，带图形界面 + 进度条 + 智能去除页码功能。把 PDF 放进任意文件夹，双击 `PDFtoJPG.exe`，点几下鼠标即可批量输出到 `<输出目录>/<PDF文件名>/page_001.jpg` 等文件中。

- 图形界面（Tkinter），双击不弹 cmd 黑窗
- 输入 / 输出目录可自由浏览选择
- 实时进度条 + 当前文件状态 + 日志窗口 + 取消按钮
- 后台线程转换，UI 不卡顿
- **去除每页页码**：智能识别（`第 N 页` / `Page N` / `1/12` / 单数字 等）+ 渲染时裁剪页脚双保险
- 支持 JPG / PNG，DPI 与 JPG 质量可选；上次的设置会自动保存到 `.pdftojpg.ini`
- 基于 [PyMuPDF (MuPDF)](https://pymupdf.readthedocs.io/) 渲染，无需 Poppler 或外部运行库
- 整个 `dist/PDFtoJPG/` 文件夹复制到任意位置即可独立运行（含 U 盘）

## 快速使用

1. 拿到 `dist\PDFtoJPG\` 整个文件夹（**不要只拿 exe**，依赖在 `_internal\` 里），放到任意位置。
2. 双击 `dist\PDFtoJPG\PDFtoJPG.exe`，主窗口弹出，启动时间约 3–5 秒。
3. 点「输入文件夹」旁的「浏览…」选 PDF 所在目录，再选输出目录（默认是 exe 同目录的 `input\` / `output\`，第一次启动会自动创建）。
4. 选格式（JPG/PNG）、DPI、JPG 质量；如需去除页码勾上「去除每页的页码（总开关）」，再独立设定「智能识别页码并抹除」/「页眉裁剪 X%」/「页脚裁剪 Y%」，三者可任意组合。
5. 点「扫描」加载 PDF 列表（显示文件名 + 页数），再点「开始转换」。进度条 **以每一页为步长** 实时推进，右侧显示当前百分比（例如 `35.0%`）；想中途停就点「取消」。
6. 完成后点「打开输出目录」即可在资源管理器里查看图片。
7. 关闭窗口时，所有参数（路径、格式、DPI、页码抹除偏好）会自动保存到 `.pdftojpg.ini`，下次启动直接复用。

> 提示：`PDFtoJPG.exe` 是 PyInstaller `--onedir` 打包的入口，配套 `_internal\` 目录里有 Tcl/Tk 与 PyMuPDF 等运行依赖。要把程序分发给别人，把整个 `PDFtoJPG\` 文件夹（含 `_internal\`）一起打包/拷贝即可。

## 输出目录约定

```
<输出目录>/
└── <PDF 原文件名 不含扩展名>/
    ├── page_001.jpg
    ├── page_002.jpg
    └── ...
```

- 文件名补零三位（`page_001` 而非 `page_1`），按文件名排序就是正确的页码顺序。
- 如果输入目录下有子文件夹，输出会**保持相同的子目录层级**。例如：

  ```
  <输入>/2025/合同.pdf  →  <输出>/2025/合同/page_001.jpg
  ```

- 重复运行不会清空 `output/`；同名图片会被覆盖。

## 去除页码功能

GUI「页码处理」面板下有 **三个互相独立** 的开关 / 数值，可以任意组合：

| 项目 | 行为 | 取值 |
| --- | --- | --- |
| **智能识别页码并抹除** | 用文本规则把识别到的页码原地抹白（不改变页面尺寸） | 复选框，开 / 关 |
| **页眉裁剪** | 渲染时从页面顶部统一裁掉指定百分比 | `0–30 %`，**0 = 不裁顶部** |
| **页脚裁剪** | 渲染时从页面底部统一裁掉指定百分比 | `0–30 %`，**0 = 不裁底部** |
| 智能扫描带宽 | 限定智能识别只在页面顶 / 底各 N% 区域内查找数字 | `5–30 %`，仅对智能识别生效 |

总开关「去除每页的页码（总开关）」一旦关闭，三个子项全部失效。

常用组合示例：

- 默认（推荐）：智能识别 + 页脚 5% — 智能识别页码 + 渲染时再裁一条窄页脚兜底。
- 同时去页眉页脚（用户文档惯有的页眉 + 页脚）：例如 **页眉 4% + 页脚 6%**，两个数值独立设定。
- 仅去页眉：页眉 5%，页脚 0；或反之。
- 文字型 PDF 嫌裁剪丢内容：仅勾「智能识别」，把裁剪都设为 0，渲染原始页面尺寸。

智能识别覆盖的常见页码格式：

| 形式 | 示例 |
| --- | --- |
| 单数字 | `1`、`23`、`156` |
| 分数式 | `1/12`、`23 / 100` |
| 中文 | `第 1 页`、`第23页` |
| 英文 | `Page 1`、`Page 1 of 12` |
| 装饰式 | `- 1 -`、`— 12 —` |
| 带标点 | `1.`、`23,` |

### 局限性

- **图片型页码无法识别**：如果 PDF 把页码烧成图像（少见），智能识别会跳过这一页 — 此时靠「裁剪页脚」兜底。
- **扫描型 PDF 无文字层**：智能识别拿不到文本，仅裁剪生效。需要正文文字识别请另用 OCR 工具，本程序不内嵌 OCR（避免单文件膨胀）。
- **正文里的页码状数字**：默认仅扫描顶/底 12% 区域，正文内的数字不会被误擦。如确需深入扫描可调高「扫描带宽」。
- 程序**不会修改原 PDF**，所有抹除操作都在内存副本上进行。

## 选项速查

| 选项 | 含义 | 推荐值 |
| --- | --- | --- |
| 格式 JPG | 体积小，最常用 | **默认** |
| 格式 PNG | 无损，文件较大 | 后期编辑选 |
| 150 DPI | 屏幕预览级 | 展示用 |
| 200 DPI | 兼顾清晰与体积 | **默认推荐** |
| 300 DPI | 印刷级 | 需打印选 |
| 450 / 600 DPI | 高清，单文件可达数 MB | 印刷出版选 |
| JPG 质量 85 | 体积最小 | — |
| JPG 质量 95 | 视觉无损 | **默认推荐** |
| JPG 质量 100 | 最高质量 | 仅必要时选 |

## 单页/单 PDF 出错怎么办

- 单页渲染失败**不会中断**整体处理；日志区会列出失败页和原因。
- 常见原因：
  - PDF 有打开密码（不支持，需先去密码）
  - 文件被其它程序占用（PDF 阅读器还开着）
  - 文件损坏
  - 输出目录不可写（权限不足 / 满盘）

## 给开发者：从源码构建

> 普通用户**不需要**这一步，直接用打包好的 `dist\PDFtoJPG\` 文件夹即可。

需要 Windows + Python 3.10+（已测试 3.13.2）。

```powershell
py -m pip install -r requirements.txt
.\build.bat
```

构建产物：`dist\PDFtoJPG\PDFtoJPG.exe`（外加同目录 `_internal\`）。

也可以手动调用：

```powershell
py -m PyInstaller --onedir --windowed --name PDFtoJPG --clean --noconfirm pdf_to_jpg.py
```

### 为什么用 `--onedir` 而不是 `--onefile`

PyInstaller 6.x + Python 3.13 + `--onefile` + `--windowed` 组合存在 Tk 顶层窗口创建后不可见的兼容问题（HWND 无法在 `EnumWindows` 中找到），同时 `--onefile` 启动需要解压约 17 秒到临时目录。`--onedir` 模式没有该 bug，启动时间从 ~17 秒降到 ~3 秒，代价是产物从单个 37 MB exe 变成 ~76 MB 的目录。如需分发给最终用户，请把整个 `dist\PDFtoJPG\` 目录打成 zip。

### 直接以脚本方式运行（开发调试）

```powershell
py pdf_to_jpg.py
```

行为与打包后一致：默认 `input/` 与 `output/` 会创建在 `pdf_to_jpg.py` 所在目录。

### 文件清单

| 文件/目录 | 作用 |
| --- | --- |
| `pdf_to_jpg.py` | 主程序源码（GUI + 转换 + 页码抹除） |
| `requirements.txt` | Python 依赖（PyMuPDF / Pillow / PyInstaller） |
| `build.bat` | 一键安装依赖 + PyInstaller `--onedir --windowed` 打包 |
| `.pdftojpg.ini` | 用户设置持久化（关窗时写入，启动时读取） |
| `input/` | 默认输入目录（首次运行自动创建） |
| `output/` | 默认输出目录（首次运行自动创建） |
| `dist/PDFtoJPG/` | 打包产物，整个文件夹一起分发 |

## 常见问题

**Q：双击 exe 之后窗口好几秒才出来？**
A：正常。PyInstaller 打包的 exe 启动时要解析 Python 运行时和加载 PyMuPDF / Tk，通常 3–5 秒，老旧机器可能更久。等待期间桌面没有任何反馈，请耐心稍候，**不要重复双击**（会启动多个实例）。

**Q：双击 exe 闪退/没反应？**
A：先用 `cmd` 打开 exe 所在目录，执行 `PDFtoJPG.exe`，看是否有报错；也可以查看 exe 同目录下是否有 `PDFtoJPG_error.log`（启动期严重错误会自动写入这个文件）。

**Q：可以只拷贝 `PDFtoJPG.exe` 吗？**
A：不能。运行时需要同目录下的 `_internal\` 文件夹（含 Tcl/Tk、PyMuPDF DLL 等）。请整个 `PDFtoJPG\` 目录一起拷贝。

**Q：可以转换有密码的 PDF 吗？**
A：当前不支持，需要先用其他工具去除密码。

**Q：去除页码后图片底部有一条很明显的白边？**
A：那是「智能识别」抹白后留下的空白。把「页脚裁剪」从 0 调到 5–10%，渲染时直接把这条页脚连同空白一起裁掉即可（智能识别可保持开启）。

**Q：去除页码不彻底，仍有数字残留？**
A：先确认 PDF 是否扫描型（无文字层）。如果有文字但识别不全：(1) 提高「智能扫描带宽」让程序往中间多扫一点；(2) 把「页脚裁剪」（或「页眉裁剪」）调到 7–10% 兜底；(3) 如果页码出现在顶部，单独把「页眉裁剪」调高，页脚保持 0。

**Q：取消按钮按下后没立即停？**
A：当前页正在渲染时无法中途打断，会在该页结束后立即停止。已经写出去的图片**不会**被回滚。

**Q：exe 可以放到没有 Python 的电脑上跑吗？**
A：可以。整个 `dist\PDFtoJPG\` 文件夹自带 Python 运行时与 MuPDF 二进制，目标机器只需 Windows 10+ 64 位。

**Q：如何加速 / 处理超大 PDF？**
A：DPI 越低速度越快、文件越小。处理 1000 页以上的 PDF 时建议先用 200 DPI 跑一份样本验证效果。
